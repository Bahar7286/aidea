## 2. Dokuz sayfa

### Sayfa 1 — AgriTwin AI
**Başlık:** Toprak bozulmadan önce gör.

| Mermi |
|------|
| Nem + lab + geçmiş → tek bakış |
| Erken uyarı, geç kriz değil |
| Çalışan web prototip |

**Konuşmacı:** “AgriTwin: sürekli ve periyodik toprak verisini içgörüye çeviren sulama odaklı dijital ikiz prototipi.”

---

### Sayfa 2 — Problem
**Başlık:** Veri var. Anlam yok.

| Mermi |
|------|
| IoT + lab + geçmiş dağınık |
| Karşılaştırma / yorum zor |
| Yıllık lab → çoğu kez geç |

**Konuşmacı:** “Çiftçi sulama değil; veriyi okuyup karşılaştırıp karar çıkaramıyor — toprak sessizce zararlı hale geliyor.”

---

### Sayfa 3 — Neden kritik?
**Başlık:** Geç fark etmek pahalı.

| Mermi |
|------|
| Nem / EC / tuz eğilimleri birikir |
| Lab yılda bir–kaç kez |
| Hasar görünmeden önce müdahale şart |

**Konuşmacı:** “Amaç yalnızca sulamak değil: bozulmayı fark etmeden önce önlemek.”

---

### Sayfa 4 — Çözüm
**Başlık:** Sürekli + periyodik → erken içgörü.

| Mermi |
|------|
| IoT nem trendi |
| Lab kimyası (tamamlayıcı) |
| Geçmiş ürün / kullanım bağlamı |
| Ortak dil: risk + öneri |

**Konuşmacı:** “AgriTwin periyotlu veri akışını analiz eder; salinite–EC–nem–lab eğilimlerinde erken uyarı üretir — hastalık teşhis ürünü değil.”

---

### Sayfa 5 — Nasıl çalışır?
**Başlık:** Dört kaynak. Bir karar.

| Mermi |
|------|
| Manuel · simüle IoT · test set · lab |
| Kural motoru + (opsiyonel) Türkçe açıklama |
| Senaryo karşılaştır → onaylı sanal sulama |

**Konuşmacı:** “Veriyi birleştiririz, riski skorlarız, sulama kararını kullanıcı onayı olmadan otomatik başlatmayız.”

---

### Sayfa 6 — Canlı prototip
**Başlık:** Şimdi ekranda. · https://aidea-three.vercel.app

| Mermi |
|------|
| Giriş: `ciftci@agritwin.demo` / `Secret12` |
| Domates Serası |
| Öneri → senaryo → onay |

**Konuşmacı:** “Canlı web MVP: nem kaynağı etiketli, lab tamamlayıcı, sulama yalnızca onayla.”

**Tıklama sırası (~60–70 sn)**

1. Siteyi göster (URL söyle).
2. Demo giriş / çiftçi hesabı.
3. **Domates Serası** aç.
4. Nem / kaynak rozeti (simülasyon etiketi görünür kalsın).
5. AI / sulama önerisi → “sulama gerekli” hikâyesi.
6. Senaryo karşılaştır (kısa).
7. Sanal sulama → **onay kapısı** göster → onayla veya “onaysız başlamaz” de.
8. (İsteğe bağlı 5 sn) Lab / malzeme: “reçete değil, bağlam.”

Yedek hesaplar: `ziraat@agritwin.demo`, `kooperatif@agritwin.demo`, `admin@agritwin.demo` — hepsi `Secret12`.

## 3. Yeniden kullanılabilir sunum prompt’u

Aşağıyı Canva / Gamma / PowerPoint / Beautiful.ai’ye yapıştır:

```

tamamlayıcı + tarım malzemesi kullanım kaydı + ürün geçmişi bağlamı. Hastalık teşhisi ürünü DEĞİL. Gübre reçetesi DEĞİL.
```
